## 一、实验目的
本实验旨在通过设计一个有限状态机（FSM）来检测特定序列"1011"，掌握数字电路中状态机的设计与实现方法。具体任务包括：
1. 绘制完整状态转移图，表示1011检测模块的状态转移；
2. 设计自动机检测1011序列，补全代码；
3. 运行Vivado仿真，分析波形图，确认状态机的运行结果。

## 二、实验原理
1. 自动机原理
  - 状态：状态机的当前状态，用于表示当前的输入和输出。
  - 输入：状态机的输入，用于触发状态转换。
  - 输出：状态机的输出，用于表示当前状态的结果。
  - 状态转换：根据当前状态和输入，将状态转换为下一个状态。

2. 有限状态自动机类型
  - Mealy型：输出不仅取决于当前状态，还取决于当前输入
  - Moore型：输出仅取决于当前状态
  - 本实验采用Moore型状态机，输出仅由当前状态决定

3. 状态编码方式
  - 二进制编码：使用最少的位数表示状态
  - 格雷码：相邻状态只有一位不同，减少状态转换时的毛刺
  - One-hot编码：每个状态用一位表示，虽然占用更多位宽，但状态转移逻辑简单，易于实现
  - 本实验采用One-hot编码，提高状态机的运行速度和可靠性

4. 状态转移设计
  - 初始状态：IDLE（空闲状态）
  - 目标序列：1011
  - 状态定义：IDLE（初始状态）、S1（检测到1）、S2（检测到10）、S3（检测到101）、S4（检测到1011）
  - 跳转逻辑：以当前输入的前序输入的最长可匹配序列为准，通过保存的状态进行记忆。
  
## 三、实验步骤
### 1. 关键代码
1. Onehot 编码补全

```verilog
parameter IDLE = 5'b00001,    // 空闲状态  
          S1   = 5'b00010,    // 检测到1
          S2   = 5'b00100,    // 检测到10
          S3   = 5'b01000,    // 检测到101
          S4   = 5'b10000;    // 检测到1011（目标序列）  
```

2. 状态转移逻辑

```verilog
always@(posedge Clk_50 or negedge Rst_n)     // 在上升沿或复位时更新状态
begin  
    if (!Rst_n) begin
        state <= IDLE;  // 复位时，状态为空闲状态
    end else begin
        case(state)     // 根据当前状态和输入数据更新状态
            IDLE: if (data_in)    // IDLE状态下输入为1时，状态转换为S1
                    state <= S1;
                else
                    state <= IDLE;
            S1: if (!data_in)    // S1状态下输入为0时，状态转换为S2
                    state <= S2;
                else
                    state <= S1;
            S2: if (data_in)     // S2状态下输入为1时，状态转换为S3
                    state <= S3;
                else
                    state <= IDLE;
            S3: if (data_in)     // S3状态下输入为1时，状态转换为S4，此时探测到1011序列
                    state <= S4;
                else
                    state <= S2;
            S4: if (data_in)
                    state <= S1;
                else
                    state <= S2;
            default: state <= IDLE;
        endcase
    end
end  
```

3. 输出逻辑

```verilog
// 当且仅当检测到目标序列1011时输出高电平 ，其余时刻输出低电平
always@(*)  
begin  
    if (state == S4)
        data_out = 1'b1;
    else
        data_out = 1'b0;
end  
```

## 五、实验思考
### 1. 遇到的问题及解决方法
1. 问题描述：输出逻辑的触发，如果在时钟上升沿触发，是否会导致输出延迟一个时钟周期？
   解决方法：采用组合逻辑（always@(*)）实现输出逻辑，这样当状态变为S4时，输出会立即变为高电平，而不需要等待时钟上升沿，提高了响应速度。
