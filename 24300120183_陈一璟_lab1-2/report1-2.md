## 一、实验目的
本实验旨在通过设计一个有限状态机（FSM）来检测特定序列"1011"，掌握数字电路中状态机的设计与实现方法。具体任务包括：
1. 绘制完整状态转移图，表示1011检测模块的状态转移；
2. 设计自动机检测1011序列，补全代码；
3. 运行Vivado仿真，分析波形，验证状态机是否能够正确检测包含重叠情况的1011序列。

## 二、实验原理
1. 自动机原理
  - 自动机是一种用于处理序列的抽象模型，能够根据输入序列进行状态转换并输出对应结果。
  - 状态：对历史输入序列的抽象，用于决定下一状态及输出
  - 输入：状态机的输入，用于触发状态转换。
  - 输出：状态机的输出，用于表示当前状态的结果。
  - 状态转换：根据当前状态和输入，将状态转换为下一个状态。

2. 有限状态自动机类型
  - Mealy型：输出不仅取决于当前状态，还取决于当前输入
  - Moore型：输出仅取决于当前状态
  - 本实验采用Moore型状态机，输出由当前状态决定。
  - 特别地，输出逻辑采用组合译码方式实现，可用连续赋值语句描述，使状态更新后输出立即响应，而不会再增加额外寄存器延迟。

3. 状态转移设计
  - 初始状态：IDLE（空闲状态）
  - 目标序列：1011
  - 状态定义：IDLE（初始状态）、S1（检测到1）、S2（检测到10）、S3（检测到101）、S4（检测到1011）
  - 跳转逻辑：以当前输入序列的最长后缀与目标序列前缀匹配为准，从而支持重叠序列的检测（如1011011）。

4. 状态编码方式
  - 二进制编码：使用最少的位数表示状态
  - 格雷码：相邻状态只有一位不同，减少状态转换时的毛刺
  - One-hot编码：每个状态用一位表示，虽然占用更多位宽，但状态转移逻辑简单，易于实现
  - 本实验采用One-hot编码，可减少状态译码的组合逻辑复杂度，在 FPGA 实现中通常有利于提高时序性能，并简化设计。


  
## 三、实验步骤
### 1. 关键代码
1. Onehot 编码补全

```verilog
parameter IDLE = 5'b00001,    // 空闲状态  
          S1   = 5'b00010,    // 检测到1
          S2   = 5'b00100,    // 检测到10
          S3   = 5'b01000,    // 检测到101
          S4   = 5'b10000;    // 检测到1011（目标序列）  
```

2. 状态转移逻辑

状态转移采用同步时序逻辑实现，复位为低有效异步复位。

```verilog
always@(posedge Clk_50 or negedge Rst_n)     // 在上升沿或复位时更新状态
begin  
    if (!Rst_n) begin
        state <= IDLE;  // 复位时，状态为空闲状态
    end else begin
        case(state)     // 根据当前状态和输入数据更新状态
            IDLE: if (data_in)    // IDLE状态下输入为1时，状态转换为S1
                    state <= S1;
                else
                    state <= IDLE;
            S1: if (!data_in)    // S1状态下输入为0时，状态转换为S2
                    state <= S2;
                else
                    state <= S1;
            S2: if (data_in)     // S2状态下输入为1时，状态转换为S3
                    state <= S3;
                else
                    state <= IDLE;
            S3: if (data_in)     // S3状态下输入为1时，状态转换为S4，此时探测到1011序列
                    state <= S4;
                else
                    state <= S2;
            S4: if (data_in)
                    state <= S1;
                else
                    state <= S2;
            default: state <= IDLE;
        endcase
    end
end  
```

3. 输出逻辑

```verilog
// 当且仅当检测到目标序列1011时输出高电平 ，其余时刻输出低电平
// 使用连续赋值的组合逻辑实现输出，使其无需等待一个时钟周期延迟即可响应
assign data_out = (state == S4);
```

## 五、实验思考
### 1. 遇到的问题及解决方法
1. 问题描述：若输出采用时序逻辑寄存，是否会导致输出相对于状态延迟一个时钟周期？    
   解决方法：采用组合逻辑实现输出，使其无需等待一个时钟周期延迟。
>
2. 问题描述：如何设计使得状态机能够处理重叠序列检测？
   解决方法：通过最长后缀匹配原则设计状态跳转，使检测到1011后根据输入跳转到S1或S2，从而支持重叠检测。

- 了解了One-hot编码的优势，以及如何通过根据输入跳转处理重叠序列的情况。
- 学习了使用Vivado进行仿真验证，观察波形结果并分析状态机的行为。
